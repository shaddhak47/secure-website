Built Secure by Default

A modern static website deployed on Netlify with industry-standard security hardening — HTTPS enforcement, strict Content Security Policy, and hardened HTTP response headers.

Explore Features

Security Features

🔐

HTTPS Everywhere

Every HTTP request is permanently redirected (301) to HTTPS via Netlify redirect rules, ensuring all traffic is encrypted in transit.

🛡️

Content Security Policy

A strict CSP header prevents cross-site scripting (XSS) and data-injection attacks by whitelisting only trusted content sources.

🚫

Clickjacking Protection

X-Frame-Options: DENY and frame-ancestors 'none' in the CSP prevent the page from being embedded in a malicious iframe.

📋

HSTS Preloading

The Strict-Transport-Security header with a one-year max-age and the preload directive ensures browsers never connect over plain HTTP.

🔍

MIME Sniffing Prevention

X-Content-Type-Options: nosniff stops browsers from interpreting files as a different MIME type, blocking a class of content-injection attacks.

🌐

Permissions Policy

Camera, microphone, geolocation, and payment APIs are explicitly disabled via the Permissions-Policy header, reducing the attack surface.

Active Security Headers

These response headers are set for every page via netlify.toml.

Header Value Purpose
Strict-Transport-Security max-age=31536000; includeSubDomains; preload Force HTTPS for 1 year
Content-Security-Policy default-src 'self'; … Prevent XSS & data injection
X-Frame-Options DENY Block clickjacking
X-Content-Type-Options nosniff Prevent MIME sniffing
Referrer-Policy strict-origin-when-cross-origin Limit referrer leakage
Permissions-Policy camera=(), microphone=(), … Disable unused browser APIs

About This Project

This website was built as part of a Cryptography & Web Security course project. It demonstrates how a simple static site can be hardened against common web threats using freely available tools and hosting features.

Deployment is handled by Netlify, which provides automatic TLS certificates via Let's Encrypt and allows fine-grained control over HTTP response headers through the netlify.toml configuration file.
